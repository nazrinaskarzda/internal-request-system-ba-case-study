# Internal Request System - Business Analysis Case Study

## Overview

This project presents a full end-to-end business analysis lifecycle for designing an internal request management system (MIS).
It demonstrates how business needs are identified, analyzed, translated into requirements, and validated through testing and evaluation.

---

## Business Problem

Internal service requests are currently handled through manual and fragmented processes, leading to delays, lack of transparency, and inefficient communication across departments.

---

## Business Objective

* Improve request processing efficiency
* Increase transparency and tracking
* Standardize internal workflows
* Enable better decision-making through data

---

## Solution Overview

A centralized internal request management system that allows employees to submit, track, and manage service requests digitally.

---

## BA Lifecycle Approach

### 1. Initiation

* Problem Statement & Business Need
* Stakeholder Identification & Analysis
* Scope Definition
* RACI Matrix

### 2. Planning & Analysis

* Elicitation Plan
* Communication Plan
* Requirements Definition
* Current State Analysis
* Traceability Matrix

### 3. Solution Design

* Functional Requirements (FR)
* Non-Functional Requirements (NFR)
* Process Models (BPMN)
* Data Models (ERD)
* Use Case & Data Flow Diagrams

### 4. Development Support

* Requirement Clarifications
* Change Management
* Ongoing Stakeholder Collaboration

### 5. Testing & Deployment

* UAT Planning
* Test Cases
* Validation of Requirements

### 6. Evaluation & Measurement

* KPI Definition
* Success Metrics
* Post-Implementation Evaluation
